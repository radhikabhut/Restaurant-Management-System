# Restaurant Management System

Restaurant Management System is a full-stack project with:

- a Go backend built with Gin and PostgreSQL
- a Flutter frontend for managing restaurant operations

The app covers authentication, menu management, user management, order workflows, and bill generation.

## Project Structure

```text
restaurant-management-system/
├── backend/   # Go API + PostgreSQL integration
└── frontend/  # Flutter client app
```

## Features

- JWT-based authentication
- Role-based access control
- Menu create, list, update, and delete
- User create, list, update, and delete
- Order create, list, update, and delete
- Bill generation
- Flutter desktop/mobile/web-ready client structure

## Backend

Location: `backend/`

### Stack

- Go
- Gin
- PostgreSQL
- JWT
- pgx connection pool

### Configuration

The backend reads configuration from environment variables.

| Variable | Default |
| --- | --- |
| `SERVER_PORT` | `8080` |
| `JWT_SECRET` | `supersecret` |
| `DB_HOST` | `localhost` |
| `DB_PORT` | `5432` |
| `DB_USER` | `postgres` |
| `DB_PASSWORD` | `password` |
| `DB_NAME` | `restaurant` |

### Run the Backend

```bash
cd backend
go mod tidy
go run .
```

The API starts on `http://localhost:8080`.

### Main API Routes

Public:

- `POST /v1/login`
- `GET /v1/ping`

Protected:

- `POST /v1/menus/list`
- `POST /v1/menus/create`
- `POST /v1/menus/update`
- `POST /v1/menus/delete`
- `POST /v1/users/create`
- `POST /v1/users/list`
- `POST /v1/users/update`
- `POST /v1/users/delete`
- `POST /v1/orders/list`
- `POST /v1/orders/create`
- `POST /v1/orders/update`
- `POST /v1/orders/delete`
- `POST /v1/orders/bill`

Protected routes require:

```http
Authorization: Bearer <token>
```

## Frontend

Location: `frontend/`

### Stack

- Flutter
- Riverpod
- HTTP package
- Shared Preferences

### Frontend API Base URL

The Flutter app currently calls:

```text
http://localhost:8080/v1
```

If you run the backend on a different host or port, update:

`frontend/lib/services/api_client.dart`

### Run the Frontend

```bash
cd frontend
flutter pub get
flutter run
```

For Linux desktop specifically:

```bash
flutter run -d linux
```

## Typical Local Setup

1. Start PostgreSQL and create a database named `restaurant`.
2. Run the backend from `backend/`.
3. Run the Flutter app from `frontend/`.
4. Log in and use the dashboard to manage menus, users, and orders.

## Notes

- The backend expects PostgreSQL to be reachable before startup.
- JWT tokens are generated by the backend and stored in the Flutter app.
- Role and permission checks are enforced by backend middleware.
